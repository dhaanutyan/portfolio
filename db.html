Database manager for merging and comparing

Design Utilities that merge 2 relational databases, saves this process in a temporary file for later editing and exporting the new database to the desired format.

    • Research

    A database is kind of related data and the way it is organised. The relational database help to track information about on example users, their names, logins information, various groups and projects they are involved and normalised it into a user table, groups table and projects table, accordingly.

    When information was being collected about a user, information stored in the optional tables would be found by searching for a special key. For instance, if the login name of a user is unique, all information about that user would be recorded with the login name as its key. Keys and table names play a "key" role when comparing or merging databases because it is for them to determine the identity of data.

    A relational database management system uses MERGE statements to write new or overwrite existing records depending on whether condition matches. Structure and tables comparing before merging. Each time software or script find differences it doesn't know what to do unless you create a rule for each situation or manage it manually. That's why It is much easier merge the databases with the same structure, especially if you try to automate the process. Lucky I have this particular case!

    In an attempt to understand what kind of errors can occur till merging I decompose the database into tables and search for various situations with unequal data:

    • The one field in the database A matches the same field in the database B;
    • The one field in the database A does not match the same field in the database B;
    • The one field does not exist in the database A but does exist in the database B;
    • The one field does not exist in the database B but does exist in the database A.

    Audience

    Developers, data architects, and others who regularly work with databases often need to locate changes, migrate differences, or synchronise versions of database tables. They need database compare and merge tool to satisfy all these requirements.

    I gather a test group of 8 programmers who have experience with databases and ask in detail what they use for this and how their process looks. What are the most common mistakes and how do they fix them. Including we collect screenshots of interfaces related to databases by comparing two databases and merging data.

    User flow

    Based on the interviews and data collected, we recreate the process of using the utility.

    First try

    The first version of layout is a step-by-step process that guides the user through the interface:

    • Step one: Loading 2 databases and specify where to save the merged one. The database can be uploaded as a file or specify a link to a remote server, enter a login and password.
    • Step two: Inspecting the structure of the loaded databases and selecting the rules and settings before merging.
    • Step three: Editing each conflict manually (If the general rules are not chosen)
    • Step four: saving and exporting the database.

    Testing

    Besides, I have a prototype I can return to my test group of programmers. They were asked to try database merging through prototype and give honest feedback. I measure the result and time of merging. The results were mostly successful but the test group was not satisfied with the user experience. Their feedback ordered to:

    • Add a number of conflicts in each table, because what this can depend on is the type of conflict.
    • Allow Apply rules for automatic merging to individual tables and to the entire database.
    • Add a preview of the table in the conflicts highlighted in it.
    • Provide an Open record for editing as a column and not as a string.
    • Remove the parameter names inside the table in the left-hand dice because they are useless and not interesting.
    • Consider that the utility will not be able to load photos from dumps
    • Remove the rule allowing the Deletion of conflicting parameters because at merge try to save a maximum of data

    We are assembling a new version. We take into account errors. Again we test on programmers and again we celebrate. All wishes exceeding the standard base interface are transferred to the second release.
